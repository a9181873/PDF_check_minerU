## Suggested Next Strategy

Next work should not blindly continue from `fa358c6`.

Recommended path:

1. Decide whether to partially revert `fa358c6` or revise it.
2. Keep or restore the previous ability to catch footer/control-number changes.
3. Consider preserving the reduced nearby text-diff merge radius, because it can prevent unrelated paragraphs, table cells, or left/right columns from collapsing into one oversized marker.
4. Do not let the reduced merge radius or small-component filtering suppress high-value footer/header changes.
5. Add a dedicated high-priority header/footer OCR pass for image-only PDFs.
6. Extract and compare meaningful patterns:
   - `Control No`
   - version dates like `2024.07`
   - document/control number strings
7. Suppress OCR garbage when text quality is low or no meaningful pattern is present.
8. For middle-page false positives, compare larger OCR text bands/paragraph zones before reporting pixel-only differences.
9. Use a two-stage visual strategy:
   - broad scan to find changed zones
   - refinement pass to split zones by text bands, table rows/cells, footer/header patterns, and local visual similarity
10. When a whole section/page is shifted, report it as structural/layout movement only if refinement cannot separate meaningful sub-differences. Do not default to one large merged marker.
11. Add regression tests for:
   - footer Control No change must be detected
   - OCR garbage should not be shown as ordinary text diff
   - visually identical or near-identical shifted regions should not be reported as real text changes
   - broad layout movement should not collapse all local changes into one marker
